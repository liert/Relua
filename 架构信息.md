# 四、为什么 Instruction 应该持有 proto？

你的问题是：

> 一个指令应该属于一个 proto，不同 proto 中的 pc 不同，那 Instruction 为什么不需要外部传入参数，而是自己解析 raw？

你应采用以下设计：

### Instruction 构造时只解析 raw → opcode/a/b/c，不依赖 proto

### 但 Instruction 保留对 proto 的弱引用（只用于 toString）

```
class Instruction {
    final int pc;
    final int raw;
    final int opcode;
    final int a, b, c, bx, sbx;

    final LuaProto proto; // 弱关联，只用于调试输出

    @Override
    public String toString() {
        // 通过 proto 拿常量池
        Constant k = proto.constants[b];
        return String.format("%d [-]: GETGLOBAL R%d K%d ; %s", pc, a, b, k.toStringRepr());
    }
}
```

这样设计的优点：

1. Instruction 保持无状态，不参与语义解析
2. 对 proto 的依赖只在 toString，不会引入循环依赖
3. 如果你在 CFG 或 IR 中复制 Instruction，不会破坏结构
4. Instruction 解析只专注“格式”，不负责“语义”
